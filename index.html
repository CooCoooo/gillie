    JavaScript MicroFramework inspired in BackboneJS

    Introduction

    Gillie is JavaScript micro-framework that lets you easily create JavaScript classes to better organize your app. Gillie can as well serve as an introductory framework to learn Backbone, given its similarities with a Backbone view. On the other hand Gillie is very easy to setup and only requires jQuery.

    All the methods you write in a Gillie class will extend the prototype object and will be shared for every instance.

    Basic class

    
var View = Gillie.extend({

        initialize: function() {
            // Do stuff...
        }

    ,   events {
            'click .my-link': 'handleClick'
        }

    ,   handleClick: function( e ) {
            e.preventDefault();

            console.log( "It works" );

        }

});

// Create instance
var myView = new View();

    Events

    The events syntax is intended to be very readable. You define your events in the events attribute, using a simple syntax: event, selector, and callback as the value of the key.

    var View = Gillie.extend({

        events {
                'click .my-link': 'handleClick'
            ,   'keyup .my-input': 'handleType'
            ,   'dbclick .second-link': 'handleDbClick'
        }

        ...
});

    Understanding `el`

    el is a class attribute that helps namespacing your selectors in order to avoid conflicts. el Can be the name of you container. In this case, if you have a container let's say .container-1, you just have to set el to that class and the events you define will be bound to elements within that container. If you don't define an el attribute, it will be set to document. 

    var View = Gillie.extend({

        initialize: function() {
            // Do stuff...
        }

        // Element to bind from
    ,   el: '.container-1'

        //  Events will be bound to .container-1 elements
    ,   events {
            'click .my-link': 'handleClick'
        }

    ,   handleClick: function( e ) {
            e.preventDefault();

            console.log( "It works" );

        }

});

    Getting data attributes

    Contrary to jQuery, in Gillie the context inside the callbacks of events is not the element that fired the event, but the class in which we're defining the events. So in order to get the element that fired the event, we have to use e.currentTarget like in this example.

    
var View = Gillie.extend({

        initialize: function() {
            // Do stuff...
        }

    ,   el: '.container-1'

    ,   events {
            'click .my-link': 'handleClick'
        }

    ,   handleClick: function( e ) {
            e.preventDefault();

            // `e` holds an attribute pointing the target of the event
            // which we can use to access that element.
            var title = $( e.currentTarget ).data( "title" );
            console.log( title );

        }

});

    Usage

    In order to start writing your app, include your script as follows.

    <script src="js/jquery.js"></script>
<script src="js/gillie.js"></script>
<script src="js/yourapp.js"></script>

    History

    I was working in a JavaScript project that had a large codebase, at the same time I was learning Backbone on my own, and was fascinated with it, one day I thought that it would be nice to use Backbone in our project, so that we can take advantage of its features.

    Even though we were using an MVC model at the project, I realized that implementing Backbone on it wouldn't be that easy, So I decided to create a small library which would give a more clear structure to our views and give us several Backbone features without requiring us to dramatically affect the codebase.

    Is in this way how Gillie was born.

    Motivation

    On the other hand, I hope Gillie to make it easy for people to get used to new JavaScript design patterns and to help them to start using MV* frameworks like Backbone, AngularJS, EmberJS and the like.
